# 交互模式

`neo` 默认进入交互式 TUI。本页覆盖输入方式、权限模式、审批对话框、流式输出，以及队列与转向（queue & steer）等高级交互。

## 输入模式

| 模式 | 触发方式 | 用途 |
| --- | --- | --- |
| 单行 | 直接输入，按 `Enter` | 简短提问与指令 |
| 多行 | `Alt+Enter` 或 `Ctrl+J` 插入换行 | 多段 prompt、代码片段 |
| 历史回溯 | 空 prompt 时按 `↑` / `↓` | 复用历史输入 |
| Shell 模式 | 进入 shell 命令编辑路径 | 编辑排队的 shell 命令 |

### 文件/路径补全

直接输入相对或绝对路径前缀即可触发本地文件补全；输入 `/` 触发斜杠命令补全；输入 `@` 触发模型别名补全。按 `Tab` 确认补全建议。

### `@` 引用文件

prompt 中使用 `@path/to/file` 会在提交时把文件内容拼进消息，免去手动复制粘贴：

```
审查 @src/parser.rs 里 error 处理是否完备
```

## 图片粘贴

Neo 支持把剪贴板里的图片作为多模态附件随 prompt 一起发送。

| 平台 | 粘贴快捷键 |
| --- | --- |
| macOS / Linux | `Ctrl+V` |
| Windows | `Alt+V` |

粘贴后会在 prompt 里插入一张图片附件预览，随下一条消息一起发送。需要具备图片能力的模型（如 `gpt-4.1`）。

## 斜杠命令

在 prompt 开头输入 `/` 即可触发命令补全。常用命令：

| 命令 | 作用 |
| --- | --- |
| `/new` `/clear` | 开始一个全新的本地会话 |
| `/resume` | 打开会话选择器 |
| `/model [alias]` | 切换模型（不带参数打开模型选择器） |
| `/provider` | 查看已配置供应商 |
| `/mcp` | 查看/管理 MCP 服务器 |
| `/tasks` | 查看后台任务，包括对话轮次运行期间 |
| `/plan [on\|off\|clear]` | 切换计划模式 |
| `/goal <objective>` | 启动/管理目标模式 |
| `/compact [instruction]` | 请求手动上下文压缩 |
| `/permissions` `/ask` `/auto` `/yolo` | 切换权限模式 |
| `/btw <question>` | 打开侧边问面板 |
| `/skill:<name>` | 激活某个技能 |
| `/help` | 打开帮助面板 |

完整列表请见 [斜杠命令参考](../reference/slash-commands.md)。

## 权限模式

权限模式决定 Neo 在调用工具时是否需要你逐次确认。可用 `/permissions` 切换或用启动 flag。

| 模式 | flag | 行为 |
| --- | --- | --- |
| **Ask**（默认） | — | 命令、编辑、其他有风险的动作都会先询问；只读/搜索工具直接放行 |
| **Auto** | `--auto` | 面向无人值守执行：工具动作自动批准，运行中的模型回合不能使用 `AskUserQuestion` |
| **YOLO** | `--yolo` | 自动批准普通工具动作。PlanMode 与 GoalMode 审阅仍会显示（与 Ask 相同）。需要时 Neo 仍可向你提问 |

> `--auto` 与 `--yolo` 互斥。Auto 适合脚本化/headless 场景；YOLO 适合你愿意放手但希望保留互动提问的工作流。

对于可能需要澄清的工作流，Neo 可以在回合开始前显示交互预检。可选澄清的工作流可以按 best-effort 假设继续；缺少必需输入的工作流会要求你先切到 Ask 或取消，避免跑到一半才停住。

在 TUI 内随时可用 `/ask` `/auto` `/yolo` 或 `Shift+Tab` 循环切换开发模式（Normal → Plan → Goal）。

## 审批对话框

当 Neo 要执行需要批准的工具调用时，会弹出审批对话框。**选项是动态的**：运行时只提供对该调用能兑现的动作。标签仅作展示；确认时提交的是请求里的类型化动作。

### 普通 Tool / Shell

| 选项 | 含义 |
| --- | --- |
| **Approve once** | 仅本次允许 |
| **会话授权**（若提供） | 本次会话内匹配的操作自动放行 |
| **前缀授权**（若提供） | 以安全 token 前缀匹配的 shell 命令自动放行 |
| **Reject** | 拒绝本次调用，Neo 会收到拒绝信号并据此调整 |

普通 Tool/Shell 审批**不提供修订反馈**。仅当存在安全的可复用范围时才会出现会话/前缀授权。

**后台 Bash** 永不提供可复用授权——只有单次 Approve 与 Reject。

### Plan / Goal 审阅

Plan 与 Goal 审阅使用同一对话框外壳，但动作集不同（详见 [计划模式](plan-mode.md) 与 [目标模式](goals.md)）。**Ask** 与 **Yolo** 会显示这些审阅；**Auto** 会跳过。

操作键：

| 键 | 动作 |
| --- | --- |
| `↑` `↓` / `PageUp` `PageDown` | 在选项间导航 |
| `Enter` | 确认当前选项（修订类动作随后收集非空反馈） |
| 数字键 | 跳到第 N 个可见选项 |
| `Esc` / `Ctrl+C` | 取消请求（若有运行中的 turn，会一并中断） |

## 流式输出

Neo 默认以流式方式渲染模型输出：助手文本边生成边打印到 transcript，工具调用与结果以可折叠卡片显示。

- `Ctrl+T`：展开/折叠 Todo 面板
- 工具输出卡片可折叠/展开
- 用 transcript 选择快捷键可选中并复制某段输出

## /btw 侧边问

`/btw` 打开一个临时的**侧边问面板**，在主会话之外问一个旁路问题。它继承当前会话的上下文，但回答不会写回主会话记录：

```
/btw 这里的 fizzbuzz 复杂度是多少？
```

| 键 | 动作 |
| --- | --- |
| `/btw <text>` | 打开面板并立即提问 |
| 空 prompt + `↑` / `↓` | 在面板历史中滚动 |
| `Esc` | 关闭面板 |

侧边问运行时主 turn 不受影响；同时只能有一个 `/btw` 在跑。

## 队列与转向（Queue & Steer）

Neo 支持在 Agent 正忙时排队后续指令，或在下一个断点注入一条转向消息。

### 排队 follow-up

当 Neo 正在执行 turn 时，直接输入新 prompt 并提交——它会被加入 **follow-up 队列**，等当前 turn 结束后自动执行。

如果 MCP 服务器仍在启动，提交的 prompt 会进入同一个待处理队列。所有已启用服务器进入终态后，Neo 会启动队首 prompt；启动失败或需要认证的服务器不会继续阻塞 prompt。

| 键 | 动作 |
| --- | --- |
| `Ctrl+S` | 把当前输入作为 **steer** 消息注入下一个断点 |
| `Alt+↑` | 把队首的 follow-up 拉回编辑器修改后再提交 |

### Steer 注入

`Ctrl+S`（`PromptSteer`）会把当前 prompt 作为**转向指令**注入正在运行的 turn，在下一个工具调用断点被 Agent 看到，用于纠正方向而不必等 turn 结束。若无运行中的 turn，则等价于排队一条 follow-up。

## 退出

| 键 | 行为 |
| --- | --- |
| `Ctrl+C` 两次 | 确认后退出（或先清空 prompt） |
| `Ctrl+D` 两次 | 同上 |
| `Ctrl+Z` | 挂起到后台（suspend） |

`Ctrl+C` 单次会先尝试：中断运行中的 turn / 拒待审批 / 关闭覆盖层 / 清空 prompt，再才是退出确认。
