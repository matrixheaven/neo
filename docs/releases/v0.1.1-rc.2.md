# Neo v0.1.1-rc.2

> Release channel: Release candidate (not stable)
>
> Tag: `v0.1.1-rc.2+20260721.0634`
>
> Date: 2026-07-21 06:34 UTC

This release candidate contains the changes from `v0.1.0` through
`v0.1.1-rc.2`. The package version remains `0.1.1`; the prerelease identifier
and timestamp identify this pre-stable distribution.

RC2 supersedes the incomplete RC1 build. RC1 produced only three of six
platform archives and remains marked as an incomplete prerelease.

## v0.1.0 Baseline

`v0.1.0` established Neo's first distributable baseline:

- Local-only Rust CLI and TUI with no hosted backend, account, or telemetry.
- OpenAI Responses, Anthropic, Google, and OpenAI-compatible providers.
- Permission-gated built-in file, shell, terminal, planning, goal, and task tools.
- Resumable and forkable workspace-scoped JSONL sessions.
- Layered skills, MCP server support, queue-and-steer interaction, and
  macOS/Linux/Windows release binaries.

The `v0.1.0` GitHub Release did not include a written changelog; this section
records that baseline explicitly for comparison.

## v0.1.1 Changes

### Highlights

- Prepared multi-file Edit and Write executions now follow a prepare, approve,
  stale-check, and commit flow instead of mutating files before approval.
- Delegate and transcript cards expose bounded, replay-safe summaries for
  prepared Edit and Write activity while preserving the existing card layout.
- Shell execution and terminal output are more inspectable and bounded, with
  ordered control input and safer live command visibility.
- Session, task, provider, and tool-runtime paths received broad reliability
  hardening across replay, cancellation, persistence, and cross-platform I/O.

### Added

- Prepared batch Edit with ordered `files[]`, `replacements[]`, and
  `expected_matches`, multi-path instruction probes, Plan-mode validation,
  typed approval data, progress events, stale rechecks, and truthful partial
  commit states.
- Prepared batch Write with multi-file authorization, approval presentation,
  bounded summaries, replay-safe Delegate details, and a documented `files[]`
  contract.
- Improved shell tool-call visibility, ordered terminal control input, and
  bounded log capture for run mode.
- Cross-platform Windows notification and durable-directory handling.

### Changed

- Resume-prefix stability now uses append-only skill catalog snapshots.
- Edit and Write previews, summaries, and transcript frames size themselves to
  their content without changing Delegate-family card semantics.
- Provider behavior now preserves Google tool-call identity and honors
  Anthropic cache retention; prompt timestamps use `chrono`.
- Dynamic workflow and terminal ordered-input design records document the
  current implementation boundaries.
- The PTY dependency returned to `portable-pty 0.9.0` to preserve Linux ARM64
  binary compatibility without raising the minimum glibc version.

### Fixed

- Hardened prepared Edit and Write authorization, stale-input detection,
  replay, and durable file synchronization, including Windows directories.
- Stabilized task-browser navigation during active turns and retained commands
  in completed Bash cards.
- Fixed duplicate transcript snapshots, bounded captured logs, bracketed paste,
  Kitty image decoding, terminal control characters, and Windows system
  notifications.
- Prevented blocking persisted-task reads, recovered stale JSON-store locks,
  stopped completed cancellation bridges, and failed child resume cleanly on
  replay errors.
- Enforced Edit write roots and classified image-generation/provider errors.
- Resolved the Rust 1.97 Clippy errors that blocked the repository CI workflow.
- Fixed Windows release builds by declaring the notification encoder as a
  Windows runtime dependency rather than a test-only dependency.
- Fixed Linux ARM64 release linking by removing the `xpty` `close_range` glibc
  symbol requirement.
- Made built-in skill refreshes atomic so concurrent runtimes cannot observe a
  truncated `SKILL.md`.

### Security and Safety

- Credential-bearing provider headers are marked sensitive.
- Edit and Write operations are authorized against workspace/write roots and
  revalidated before commit.
- Shell and terminal output/control paths have bounded capture and sanitization
  behavior to reduce truncation, injection, and resource-exhaustion risk.
- Windows notification text is base64-encoded before insertion into the
  PowerShell command, preventing untrusted text from becoming script syntax.

### Verification

- `cargo fmt --all --check`
- `cargo clippy --workspace --all-targets --all-features -- -D clippy::all`
- `cargo build -p neo-agent`
- `cargo nextest run --workspace --all-features`
- Release builds for Linux x86_64/ARM64, macOS x86_64/ARM64, and Windows
  x86_64/ARM64.

## Known RC Status

This is not a stable release. Interfaces, provider behavior, TUI presentation,
and persistence details may still change before `v0.1.1`. Do not treat this RC
as a compatibility guarantee for production automation.
